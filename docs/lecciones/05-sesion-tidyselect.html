Elige tu(s) columna(s)

En la sesión pasada vimos cómo usar filter para elegir o filtrar filas. Ahora veremos cómo usar select para elegir columnas. Si bien ya sabemos cómo elegir columnas (de un dataframe, con el símbolo de $, o con el uso del paréntesis recto: miDataFrame[filas,columnas]), select nos hace la sintaxis más fácil y más legible, pero además nos permite combinarla con otras funciones que veremos más adelante, lo que nos da mucha más potencia.

El paquete tidyselect

A menudo nos enfrentamos a conjuntos de datos con muchas columnas. Muuuchas. O sea, de repente tenemos 10 o 15 columnas que queremos elegir para algo, del total de 32 columnas que tenemos. Y a veces no son columnas contiguas (de la 5 a la 15, por ejemplo), así que llamarlas por nombre o por número es igual de engorroso: tendremos que escribir a mano los 10 o 15 números o nombres de columna a mano…

tidyselect al rescate!! Este pequeño paquete (parte del tidyverse obvvviamente), nos provee de varias funciones que nos hacen la vida más fácil a la hora de elegir varias columnas a la vez, siempre y cuando tengan alguna REGULARIDAD en los nombres. Por ejemplo, todas empiezan o terminan con la misma expresión, o tienen algo en común en su nombre (y generalmente éste suele ser el caso). Por ejemplo, puedo tener medidas “pre” y “post” de 10 escalas psicométricas, quizás se llamen algo así como “puntuacion_pre_UnaEscala”, “puntuacion_post_otraEscala”, etc. Éste es el escenario típico en el que las siguientes funciones son más útiles. Como es el espíritu tidy, los nombres son bastante intuitivos y nos permiten inferir qué cornos hacen:

  • starts_with() Elige columnas que empiecen con el mismo prefijo.
  • ends_with() ” ” ” terminen con el mismo sufijo.
  • contains() ” ” ” contengan la misma expresión en su nombre.
  • matches() ” ” cuyo nombre dé match con una expresión regular (como contains pero avanzado, para “hackers” digamos).
  • num_range() ” ” cuyo nombre sea consistente con un esquema numérico (“toma_1”, “toma_2”, etc.)

Estas funciones suelen usarse como ayudantes, digamos, cuando usamos otras funciones como select. Veamos cómo usarlas entonces.

Across the tidyverse

Todo muy lindo, pero esto se puede usar para algo más…. potente? Bueno, así como group_by sirve para agrupar filas, típicamente para sumarizar o mutar según estas agrupaciones, se puede hacer algo parecido pero con las columnas. Es decir, se puede sumarizar y mutar, aplicando la misma función (o funciones) a diferentes columnas. Qué pasa si quiero tener la media y el desvío de cada columna que sea numérica, por ejemplo. Digamos que quiero pasar a z-score todas mis medidas, que tengo, convenientemente, en diferentes columnas. Cómo puedo hacer eso? Con la función across.

datosLimpios %>% summarize(across(ends_with("TOTAL"),mean,na.rm=T))
  NPI16TOTAL GSESTOTAL  ADTOTAL  SDTOTAL
1   3.233333  26.65806 15.75238 28.39032

EH?

Vamos por partes. La función across necesita de al menos dos argumentos: un conjunto de columnas, y una función.

across(ends_with("TOTAL"),mean)

Pero como nos advierte R en la consola si queremos correr lo que se ve arriba directamente, across no es una función que pueda usarse “suelta”, sino que tiene que ir dentro de un verbo de dplyr: (generalmente summarize, mutate, o el rara vez usado transmute, que no vimos).

Lo que hace across es indicarle a mutate o summarize, que la operación (cuenta, suma, promedio, etc) se va a hacer a varias columnas a la vez. Por eso hay que indicarle a cuáles columnas. En este caso estamos usando tidyselect para indicarle que son las columnas que terminan en “TOTAL” las que queremos. El otro argumento es la función que queremos usar. En este caso, queremos la media, por eso indicamos la función mean, así, sin paréntesis ni nada.

Entonces tendríamos:

datosLimpios %>%  #pongo los datos en el pipe
  summarize( #sumarizo
    across( # varias columnas a la vez
      ends_with("TOTAL"), # las que terminan en "TOTAL"
      mean #calculale la media a cada una
    ) 
  )
  NPI16TOTAL GSESTOTAL ADTOTAL SDTOTAL
1   3.233333        NA      NA      NA

Oops! Tenemos varios NAs! Tendría que pasarle a mean el argumento na.rm=T… Fácil, le paso ese argumento a across, para que cuando “llame” a la función mean para calcular la media de cada columna, le “pase” ese argumento. Entonces tenemos la expresión final (o sea, la que mostré al principio :P, pero acá la escribimos un poco “desaparramada” para que se vea mejor qué hace cada parte):

datosLimpios %>%  #pongo los datos en el pipe
  summarize( #sumarizo
    across( # varias columnas a la vez
      ends_with("TOTAL"), # las que terminan en "TOTAL"
      mean,  #calculale la media a cada una
      na.rm=T #pasale el argumento na.rm=T a la función .mean.
    )
  )
  NPI16TOTAL GSESTOTAL  ADTOTAL  SDTOTAL
1   3.233333  26.65806 15.75238 28.39032

Claro está, puedo elegir las columnas como yo quiera. Las siguientes expresiones hacen lo mismo que la anterior:

datosLimpios %>%  #pongo los datos en el pipe
  summarize( #sumarizo
    across( # varias columnas a la vez
      3:6, # las columnas 3 a la 6
      mean,  #calculale la media a cada una
      na.rm=T #pasale el argumento na.rm=T a la función .mean.
    )
  )
datosLimpios %>%  #pongo los datos en el pipe
  summarize( #sumarizo
    across( # varias columnas a la vez
      c("NPI16TOTAL","GSESTOTAL","ADTOTAL","SDTOTAL"), # lista de nombres de columna
      mean,  #calculale la media a cada una
      na.rm=T #pasale el argumento na.rm=T a la función .mean.
    )
  )

Como siempre, across puede integrarse en un pipeline o tubería más grande. Por ejemplo, puedo además agrupar or filas:

datosLimpios %>%  #pongo los datos en el pipe
  group_by(Sex) %>% #agrupo filas por Sexo
  summarize( #sumarizo
    across( # varias columnas a la vez
      ends_with("TOTAL"), # las que terminan en "TOTAL"
      mean,  #calculale la media a cada una
      na.rm=T #pasale el argumento na.rm=T a la función .mean.
    )
  )

Varias funciones a la vez

Ya lo dice el dicho: el que calcula la media, quiere calcular también el desvío. ¿Cómo hacerlo eficientemente con across? Simplemente agregar la función sd en la lista de argumentos (combinándolas en una lista de funciones mediante el combine c()):

datosLimpios %>%  #pongo los datos en el pipe
  summarize( #sumarizo
    across( # varias columnas a la vez
      ends_with("TOTAL"), # las que terminan en "TOTAL"
      c(mean,sd),  #calculale la media y el desvío a cada una
      na.rm=T #pasale el argumento na.rm=T a la función .mean.
    )

Hay una función que te va a ayudar: is.numeric. Si le pasás un vector, te dice si es numérico o no, como dice el nombre.

is.numeric(datosLimpios$Sex)

is.numeric(datosLimpios$NPI16TOTAL)
[1] TRUE

Entonces podemos usar esta función (o cualquier otra expresión que nos devuelva un lógico para cada columna) para seleccionar columnas en across:

datosLimpios %>% 
  group_by(Sex) %>% 
  mutate(
    across(
      is.numeric, #va sin paréntesis aquí también
      c(z=scale), #no te olvides de usar el combine
    ) 
  )
